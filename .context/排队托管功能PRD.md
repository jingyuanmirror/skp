## 1. 功能概述

排队托管功能让用户通过一句自然语言即可完成餐厅或热门品牌的排队取号，由AI Agent在后台代为持有号牌。排队期间用户可在商场内自由活动，Agent在预计到号前主动提醒，彻底释放用户在排队处的被动等待时间。

**核心价值**：将传统"站在门口等叫号"的被动等待，变为"AI代排 + 主动提醒"的托管模式，释放用户在商场内的闲逛时间。

## 2. 用户故事

| # | 角色 | 故事 | 验收标准 |
|---|------|------|----------|
| US-1 | 会员 | 我说"帮我排一个新荣记3人位"，管家帮我取号并托管排队 | Agent回复确认 + 弹出排队卡片（排队号、前方等待、预计时长） |
| US-2 | 会员 | 排队快到号时，管家主动提醒我 | 前方≤2组时Agent主动发消息"即将到号"；到号时再发"已到号" |
| US-3 | 会员 | 我随时问排队进度，管家告诉我当前状态 | 输入"排队排到哪了"→ Agent展示当前排队卡片 |
| US-4 | 会员 | 我没有排队记录时问排队状态，管家引导我取号 | 提示可排队店铺 + 快捷回复 |
| US-5 | 会员 | 品牌专柜也能排队托管 | "帮我排Chanel" → 同样走排队托管流程 |
| US-6 | 会员 | 排队人数默认2人，但我也可以指定 | "帮我排新荣记3人位" → 3人位；"帮我排新荣记" → 默认2人位 |

## 3. 交互流程

### 3.1 排队托管主流程

```
用户：帮我排一个新荣记3人位
  ↓
Agent：[意图识别 → 排队托管]
  ↓
Agent 回复：「李先生，已为您在新荣记（5F）托管排队，3人位，
            取号A38。前方等候5组，预计约需40分钟。
            到号前我会提前提醒您，您可以在商场内自由逛逛。」
  ↓
Agent 弹出排队卡片
  - 状态：🟢 排队中
  - 排队号：A38
  - 前方等待：5组
  - 预计约40分钟，到号前会提前提醒

快捷回复：[查询排队进度] [今日专属优惠] [查询停车状态]
```

### 3.2 即将到号提醒（自动触发）

```
（约24秒后，前方等待≤2组）

Agent 自动发送：「李先生，您在新荣记的排队即将到号，
                前方仅剩2组，请准备前往5F。」

Agent 弹出排队卡片（状态更新）
  - 状态：🟠 即将到号
  - 前方等待：2组

快捷回复：[导航到店铺] [查看菜单]
```

### 3.3 已到号提醒（自动触发）

```
（再约24秒后，前方等待=0组）

Agent 自动发送：「李先生，您在新荣记的排队已到号！
                请前往5F入座，过号需重新排队哦。」

Agent 弹出排队卡片（状态更新）
  - 状态：🟡 已到号
  - 前方等待：—
  - 请尽快前往店铺，过号需重新排队

快捷回复：[导航到店铺] [今日专属优惠]
```

### 3.4 排队进度查询（有记录）

```
用户：排队排到哪了 / 我的排队怎么样
  ↓
Agent：展示当前排队进度卡片 + 文案说明

（文案根据状态动态变化：排队中/即将到号/已到号）
```

### 3.5 排队进度查询（无记录）

```
用户：排队排到哪了
  ↓

      您可以说"帮我排一个新荣记3人位"，
      我会为您托管排队并在到号时提醒您。」

快捷回复：[帮我排新荣记] [帮我排Chanel] [今日专属优惠]
```

## 4. 意图识别规则

### 4.1 排队托管意图

用户消息命中"帮我排/排一个/排号/取号"模式，且能解析出店铺名：

**正则表达式**：

含人数：`/(?:帮我排|排一个|帮我排队|排号|取号)[\s]?(.{1,10}?)[\s]?(\d)\s*人位?/`
不含人数：`/(?:帮我排|排一个|帮我排队|排号|取号)[\s]?(.{1,10}?)(?:的|$)/`

**解析规则**：

| 输入 | 解析结果 |
|------|----------|
| 帮我排一个新荣记3人位 | 店铺=新荣记, 人数=3 |
| 帮我排新荣记 | 店铺=新荣记, 人数=2（默认） |
| 帮我排Chanel | 店铺=Chanel, 人数=2（默认） |
| 取号新荣记4人位 | 店铺=新荣记, 人数=4 |

**店铺名匹配**：用户输入的店铺名与 `QUEUE_VENUES` 数据库中的 key 做 includes 双向匹配。

### 4.2 排队查询意图

| 关键词 | 示例 |
|--------|------|
| 排队排到 | "排队排到哪了" |
| 排到了 | "排到了吗" |
| 排队进度 | "排队进度怎么样" |
| 排队怎么样 | "排队怎么样了" |
| 排到哪 | "排到哪了" |
| 排队状态 | "排队状态" |
| 我的排队 | "我的排队" |
| 排队号 | "我的排队号" |

### 4.3 意图路由优先级

```
1.  品类偏好回复（入会后）
2.  入会意图
3.  会员咨询意图
4.  排队托管意图  ← 本功能
5.  排队查询意图  ← 本功能
6.  品牌排队查询（排队时长 + 交叉发券）
7.  品牌券查询
8.  通用券查询
9.  权益场景引导入会
10. 停车位置录入
11. 停车状态查询
12. 兜底通用回复
```

> **注意**：排队托管（4）优先于品牌排队查询（6）。用户说"帮我排Chanel"走托管流程，说"Chanel排队多久"走时长查询+交叉发券流程。

## 5. 排队状态流转

### 5.1 状态模型

```
queuing ──(前方≤2)──→ almost ──(前方=0)──→ ready
```

| 状态 | 含义 | 卡片标签 | 标签颜色 | 底部提示 |
|------|------|----------|----------|----------|
| queuing | 正在排队 | 排队中 | 🟢 绿色 | 预计还需约X分钟，到号前会提前提醒您 |
| almost | 即将到号 | 即将到号 | 🟠 橙色 | 预计还需约X分钟，到号前会提前提醒您 |
| ready | 已到号 | 已到号 | 🟡 金色 | 请尽快前往店铺，过号需重新排队 |

### 5.2 模拟推进逻辑（Demo阶段）

- 每12秒自动减少1组前方等待
- 每组等待预估8分钟
- 状态变更时自动发送Agent提醒消息 + 更新排队卡片

### 5.3 提醒机制

| 事件 | 触发条件 | Agent 行为 |
|------|----------|-----------|
| 即将到号提醒 | ahead ≤ 2 且之前未提醒 | 发送消息：「即将到号，前方仅剩X组，请准备前往」+ 排队卡片(almost) |
| 到号提醒 | ahead = 0 且之前未提醒 | 发送消息：「已到号！请前往入座，过号需重新排队」+ 排队卡片(ready) |

> 提醒去重：通过 `queueNotifiedRef` 确保每个状态只提醒一次，避免重复通知。

## 6. 数据模型

### 6.1 前端状态

```typescript
interface QueueInfo {
  brand: string;       // 店铺名
  floor: string;       // 楼层
  partySize: number;   // 人数
  queueNo: string;     // 排队号
  ahead: number;       // 前方等待组数
  estMin: number;      // 预估等待分钟
  enrolledAt: number;  // 托管时间戳
  status: "queuing" | "almost" | "ready";
}

interface QueueCard {
  type: "queue-card";
  brand: string;
  floor: string;
  partySize: number;
  queueNo: string;
  ahead: number;
  estMin: number;
  status: "queuing" | "almost" | "ready";
}
```

### 6.2 可排队店铺数据库（Demo）

| key | 店铺名 | 楼层 | 类型 |
|-----|--------|------|------|
| 新荣记 | 新荣记 | 5F | 餐饮 |
| 大董 | 大董烤鸭 | 4F | 餐饮 |
| flair | FLAIR 高端茶饮 | 4F | 咖啡茶饮 |
| chanel | Chanel 精品店 | 1F-A12 | 品牌专柜 |
| 香奈儿 | Chanel 精品店 | 1F-A12 | 品牌专柜 |
| hermes / 爱马仕 / hermès | Hermès | 1F-B03 | 品牌专柜 |
| lv / 路易威登 | Louis Vuitton | 1F-C01 | 品牌专柜 |
| dior / 迪奥 | Dior | 1F-D05 | 品牌专柜 |
| gucci / 古驰 | Gucci | 1F-E08 | 品牌专柜 |

> 支持中英文、大小写模糊匹配。

### 6.3 排队号生成规则（Demo）

格式：`{大写字母A-H}{10-99随机两位数}`，如 `A38`、`C52`。

### 6.4 等待时间预估（Demo）

- 前方初始等待组数：3~8组（随机生成）
- 每组预估等待时长：8分钟
- 总预估 = ahead × 8

## 9. 与主 PRD 的关联

本功能对应《北京SKP AI私享管家系统产品需求文档 v1.0》§2.4.2 全功能排队托管，当前实现为**基础排队托管 + 到号提醒**的 Demo 版本。

### 与主 PRD 的差异说明

| 主 PRD 要求 | 当前实现 | 后续规划 |
|-------------|---------|---------|
| 对接商场餐饮/品牌排队系统API | ❌ 前端硬编码14个店铺 | M2 对接商场排队系统 |
| 排队时长实时获取 | ❌ 随机生成 | M2 对接实时排队系统 |
| 语音提醒到号 | ❌ 仅对话内消息提醒 | M2 微信服务号+语音提醒 |
| 多店铺同时排队 | ❌ 仅支持单个排队 | M2 多队列并行管理 |
| 过号处理（重新取号/顺延） | ❌ 仅文案提示 | M2 对接排队系统过号逻辑 |
| 排队数据持久化 | ❌ 会话级，刷新丢失 | M2 后端持久化 |
| "导航到店铺"快捷回复 | ❌ 占位按钮 | M2 室内导航集成 |
| "查看菜单"快捷回复 | ❌ 占位按钮 | M2 餐厅菜单对接 |
