# 北京SKP AI私享管家 — 智能发券功能 PRD

> **模块归属**：模块三 · 动态权益与卡券策略引擎 (Smart Couponing)
> **文档版本**：v1.0
> **创建日期**：2026-06-27
> **当前状态**：已实现（Demo 阶段）

---

## 1. 功能概述

智能发券功能让 Agent 根据用户意图和场景上下文，主动推送个性化优惠券。支持三种发券模式：

1. **通用券查询** — 用户直接询问优惠/券时，推送商场通用券
2. **品牌券查询** — 用户询问特定品牌的优惠时，推送品牌专属券 + 通用券
3. **场景交叉推荐** — 用户在排队等特定场景下，Agent 主动推送跨品类交叉券（如排队久→推荐咖啡券）

**核心价值**：告别静态"领券中心"，实现场景感知的动态即时发券，在用户最需要时提供最相关的权益。

## 3. 交互流程

### 3.1 通用券查询

```
用户：有没有优惠券 / 有什么优惠
  ↓
Agent：李先生，目前以下商场通用优惠券可供领取，您可以直接使用：
  ↓
Agent 弹出券卡片（商场通用券 × N）

```


## 4. 意图识别规则

### 4.1 通用券查询

**匹配规则**：消息包含优惠关键词且不包含入会/会员意图

| 关键词 | 示例 |
|--------|------|
| 优惠券 | "有没有优惠券"、"给我一张券" |
| 有没有券 | "有没有券"、"有券吗" |
| 优惠 | "有什么优惠"、"优惠活动" |
| 折扣 | "有折扣吗"、"打折吗" |
| 打折 | "今天打折吗" |
| 活动 | "有什么活动"、"促销活动" |
| 促销 | "促销信息" |
| 发券 | "给我发张券" |
| 领券 | "我想领券" |

**正则表达式**：`/优惠券|有没有券|优惠|折扣|打折|活动|促销|发券|领券/`

**排除条件**：不匹配入会意图（`isMembershipIntent`）

### 4.2 品牌券查询

**匹配规则**：消息同时包含品牌关键词 + 优惠关键词

| 品牌关键词 | 优惠关键词 |
|-----------|-----------|
| Chanel / 香奈儿 | 优惠券 / 有券吗 / 优惠 / 折扣 / 打折 / 活动 / 促销 |
| Hermès / 爱马仕 | 同上 |
| Dior / 迪奥 | 同上 |
| Gucci / 古驰 | 同上 |
| Louis Vuitton / LV / 路易威登 | 同上 |
| Cartier / 卡地亚 | 同上 |
| La Mer / 海蓝之谜 | 同上 |

### 4.3 排队场景交叉推荐

**匹配规则**：消息同时包含品牌关键词 + 排队关键词

**排队关键词**：`排队|等多|排多久|排队多久|人多人|要等多`

**交叉推荐触发条件**（全部满足）：
1. 匹配到品牌排队数据
2. 排队等待时间 ≥ 30分钟

**推荐策略**：

| 排队时长 | Agent 行为 | 推荐券 |
|----------|-----------|--------|
| ≥ 30分钟 | 告知时长 + 推荐歇脚地点 + 发券 | 仅 FLAIR 咖啡券（交叉推荐场景不叠加品牌券/通用券） |
| 15~30分钟 | 告知时长 + 提供托管排队选项 | 无 |
| < 15分钟 | 仅告知时长 | 无 |

## 6. 券卡片 UI 设计

### 6.1 视觉规格

| 属性 | 值 |
|------|-----|
| 卡片宽度 | max 300px |
| 圆角 | 14px |
| 主背景 | 白色 #FFFFFF |
| 边框 | 1px solid rgba(184,146,74,0.15) |
| 顶部装饰条 | 商场通用：金色渐变（#B8924A → #D4B978）；品牌专属：金绿渐变（#C9A86C → #8BA888） |
| 弹出动画 | 从底部滑入 + 缩放：`opacity: 0→1, y: 16→0, scale: 0.95→1`，0.4s 弹性缓动 |

### 6.2 结构层次

```
┌──────────────────────────────────────┐
│ ═══ 金色/双色渐变装饰条 3px ═══      │
│                                      │
│  Chanel                    [品牌专属] │
│                                      │
│  8.5折   精品店专属券                 │
│                                      │
│  有效期至 2026.07.15                 │
│                                      │
│ ○                                 ○  │ ← 左右缺口装饰
└──────────────────────────────────────┘
```

### 6.3 设计要点

- **顶部装饰条**：3px 渐变色条，商场通用券用金色调、品牌专属券用金绿双色调，一眼区分
- **品牌名**：12px 加粗，黑色
- **范围标签**：右上角 8px 小 pill — 商场通用（金底金字）/ 品牌专属（绿底绿字）
- **折扣数字**：22px DM Mono 加粗金色（#B8924A），视觉焦点
- **券标题**：12px 加粗黑色，紧跟折扣数字
- **有效期**：9px 浅灰色，底部对齐
- **左右缺口**：半圆形缺口装饰模拟实体优惠券撕裂线
- **多券垂直堆叠**：当消息附带多张券时，券卡片垂直排列，每张独立弹出

## 9. 与主 PRD 的关联

本功能对应《北京SKP AI私享管家系统产品需求文档 v1.0》§2.3 模块三：动态权益与卡券策略引擎。

### 与主 PRD 的差异说明

| 主 PRD 要求 | 当前实现 | 后续规划 |
|-------------|---------|---------|
| 热门排队等待 → 精准下发咖啡券 | ✅ 已实现（排队≥30分钟触发） | M1 对接实时排队系统 |
| 跨品类连带推荐 | ✅ 已实现（排队场景跨品牌推荐） | M2 基于CRM画像自动推荐 |
| 离店临界挽回 → 晚间餐饮券 | ❌ 未实现 | M2 位置感知触发 |
| 券的核销状态同步 | ❌ 未实现（Demo展示即发放） | M1 对接POS/券包系统 |
| 排队时长实时获取 | ❌ Demo硬编码 | M1 对接商场排队系统API |
| 券数据从后台动态获取 | ❌ 前端硬编码11张券 | M1 对接营销发券API |
| 券的有效期动态计算 | ❌ 写死日期 | M1 从后端获取 |
