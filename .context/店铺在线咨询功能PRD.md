> **模块归属**：模块四 · 进阶私享管家服务 → 新增 · 来店前咨询
> **文档版本**：v1.0
> **创建日期**：2026-06-29
> **当前状态**：方案规划中

---

## 1. 功能概述

店铺在线咨询功能让用户在来店前即可通过自然语言咨询品牌信息，包括品牌位置、主营品类、当季新品、SA 联系等。Agent 基于品牌知识库 + 用户画像偏好，提供个性化的品牌推荐与信息查询，降低用户到店决策门槛。

**核心价值**：将"来不来 SKP"的决策点变为数字触点，通过品牌咨询引导预约 → 预留车位 → 到店消费，完成来店前动线闭环。

| # | 角色 | 故事 | 验收标准 |
|---|------|------|----------|
| US-1 | 潜在客户 | 我问"SKP 有没有 Moncler"，管家告诉我品牌位置和品类 | Agent 回复品牌楼层、品类信息，弹出品牌卡片 |
| US-2 | 潜在客户 | 我问"Chanel 有什么新款包"，管家告诉我当季新品 | Agent 回复新品信息 + 品牌卡片 |
| US-3 | 潜在客户 | 我说"我想买个围巾送人"，管家根据偏好推荐品牌 | Agent 基于画像推荐 2-3 个品牌 + 品牌卡片 |
| US-4 | 会员 | 我问完品牌信息后，管家主动问我要不要预约 | 咨询完自动提供"帮我预约档期"快捷回复 |

| US-6 | 会员 | 我想联系某个品牌的 SA | Agent 提供 SA 连接信息 / "帮您安排专属 SA"快捷回复 |

## 3. 交互流程

### 3.1 品牌信息查询

```
用户：SKP 有没有 Moncler？/ Chanel 在几楼？
  ↓
Agent：[意图识别 → store-consult]
  ↓
Agent 回复：「李先生，Moncler 位于 SKP 2F-D08，主营高端羽绒服与户外时尚服饰。
            本季已上新 Mountain 系列轻量羽绒服，适合日常通勤与旅行穿搭。」
  ↓
Agent 弹出品牌卡片
  - 品牌：Moncler

  - 品类：高端户外时尚
  - 亮点：Mountain 系列上新

快捷回复：[帮我预约档期] [帮我预留车位] [联系专属SA]
```

### 3.2 新品/到货咨询

```
用户：Chanel 有什么新款包？/ Hermès 到了什么新品？
  ↓
Agent：[意图识别 → store-consult（新品子场景）]
  ↓
Agent 回复：「李先生，Chanel（1F-A12）本季新品已到店，包括：
            • Classic Flap 小号 — 小羊皮黑色金扣

            • 22 Bag 小号 — 亮片粗花呢
            数量有限，是否需要为您安排专属 SA 预约试看？」
  ↓
Agent 弹出品牌卡片（标注"本季新品"标签）

快捷回复：[帮我预约Chanel] [帮我预留车位] [看看Hermès新品]
```

### 3.3 个性化推荐（基于用户画像）

```
用户：我想买个围巾送人 / 推荐个适合送礼的品牌
  ↓
Agent：[意图识别 → store-consult（推荐子场景）+ 读取 userProfile]
  ↓
Agent 回复：「李先生，根据您的偏好，为您推荐以下送礼选择：
            1. Hermès（1F-B03）— 经典 90cm 真丝斜纹围巾，花色丰富，是经典送礼之选

            如需我为您安排试看或预约 SA，随时告诉我。」
  ↓

快捷回复：[帮我预约Hermès] [帮我预约LV] [看看更多推荐]
```

### 3.4 非会员咨询 → 会员引导

```
用户：Chanel 有什么新款包？
  ↓
Agent：[意图识别 → store-consult + 检测 !isMember]
  ↓
Agent 回复：「李先生，Chanel（1F-A12）本季新品已到店，Classic Flap 系列有多款可选。
            另外提醒您，SKP 黑卡会员可享品牌新品优先预览权与专属 SA 预约通道，
            是否需要为您办理入会？」
  ↓
Agent 弹出品牌卡片

快捷回复：[我想入会] [帮我预约Chanel] [稍后再说]
```

## 4. 意图识别规则

### 4.1 store-consult 意图

**分类策略**：由 LLM 意图分类器识别，以下模式作为辅助信号：

| 子意图 | 匹配模式 | 示例 |
|--------|---------|------|
| 品牌信息查询 | 品牌关键词 + 位置/品类疑问 | "Chanel 在几楼"、"SKP 有没有 Moncler" |
| 新品咨询 | 品牌关键词 + 新品/新款/到货 | "Chanel 有什么新款包"、"Hermès 到了什么新品" |
| 个性化推荐 | 送礼/推荐/推荐品牌 + 无具体品牌 | "推荐个适合送礼的品牌"、"我想买个围巾送人" |
| SA 连接 | 品牌 + SA/导购/顾问 | "帮我联系 Chanel 的 SA"、"有专属顾问吗" |

### 4.2 intentDescription（供 LLM 分类器使用）

```
store-consult: 用户询问SKP品牌店铺信息、品牌位置、当季新品、到货咨询、礼品推荐、联系SA导购等来店前咨询场景
```

### 4.3 与 service-qa 的区分

| 维度 | store-consult | service-qa |
|------|--------------|-----------|
| 知识域 | 品牌信息、新品、推荐 | 商场服务、餐饮、退换货 |
| 关键信号 | 包含品牌名/品类关键词 | 包含服务类关键词 |
| 数据源 | `brand-catalog.md` | `mall-knowledge.md` |
| 返回 | `BrandCard` | 纯文本 |

**路由优先级**：品牌关键词命中 → `store-consult` 优先；服务关键词命中 → `service-qa` 优先；两者都不明确 → LLM 分类器裁决。

## 5. 数据模型

### 5.1 品牌知识库（`brand-catalog.md`）

结构化 markdown，按品牌分章节：

```markdown
## Chanel
- 楼层：1F-A12
- 品类：高级成衣、皮具、鞋履、配饰、香水美妆
- 风格标签：经典优雅、法式奢华
- 当季亮点：Classic Flap 新色到店、22 Bag 限定粗花呢
- SA 预约：支持（建议提前 1 天预约）

## Hermès
- 楼层：1F-B03
...
```

### 5.2 BrandCard 数据结构

```typescript
interface BrandCard {
  type: "brand-card";
  brand: string;          // 品牌名，如 "Chanel"
  floor: string;          // 楼层位置，如 "1F-A12"
  categories: string[];   // 品类标签，如 ["皮具", "成衣", "配饰"]
  highlight?: string;     // 当季亮点，如 "Classic Flap 新色到店"
  tag?: string;           // 附加标签，如 "本季新品"、"新品预约"
}
```

### 5.3 AgentResponse 扩展

```typescript
interface AgentResponse {
  text: string;
  quickReplies?: string[];
  card?: MemberCard;
  brandCard?: BrandCard;       // 新增：品牌咨询卡片
  brandCards?: BrandCard[];    // 新增：多品牌推荐场景
  parkingCard?: ParkingCard;
  coupons?: CouponCard[];
  queueCard?: QueueCard;
  sideEffects?: AgentSideEffects;
}
```

### 5.4 Message 类型扩展

```typescript
interface Message {
  // 现有字段...
  brandCard?: BrandCard;
  brandCards?: BrandCard[];
}
```

### 6.1 视觉规格

| 属性 | 值 |
|------|-----|

### 6.2 结构层次

```
┌──────────────────────────────────────┐
│ Header（渐变背景 #F7F0E4 → #F0E8D4）│
│  🏬 品牌咨询            [本季新品]   │
├──────────────────────────────────────┤
│ Chanel                               │
│ 📍 1F-A12                            │
│                                      │

│                                      │
│ ✦ Classic Flap 新色到店              │
```

### 6.3 设计要点

- **头部**：暖色渐变 + 「商会 品牌咨询」标识 + 可选标签（本季新品/限量/预约中）
- **品牌名**：Cormorant 18px 加粗，深色
- **楼层位置**：📍 图标 + 灰色文字
- **品类标签**：9px 圆角 pill，半透明金底金字，flex-wrap 横排
- **亮点区**：✦ 金色符号 + 浅灰背景区域，展示当季亮点或新品摘要

## 7. 快捷回复策略

| 场景 | 快捷回复按钮 |
|------|-------------|
| 品牌信息查询 | [帮我预约档期] [帮我预留车位] [联系专属SA] |
| 新品咨询 | [帮我预约{品牌}] [帮我预留车位] [看看{关联品牌}新品] |
| 个性化推荐（多个品牌） | [帮我预约{品牌1}] [帮我预约{品牌2}] [看看更多推荐] |
| 非会员咨询（含引导） | [我想入会] [帮我预约{品牌}] [稍后再说] |

### 7.1 联动快捷回复的动态生成

快捷回复中的 `{品牌}` 需要根据当前咨询的品牌动态替换，实现 skill 间的丝滑跳转：

- "帮我预约 Chanel" → 触发 `appointment` skill
- "帮我预留车位" → 触发 `parking` 预留功能
- "我想入会" → 触发 `membership` skill

## 8. Skill 间联动设计

### 8.1 store-consult → appointment

品牌咨询完成后，用户表达试看/购买意愿时，自动引导预约：

```
Agent 文案追加：「是否需要为您在{品牌}安排专属 SA 预约试看？」
快捷回复追加：[帮我预约{品牌}]
```

### 8.2 store-consult → parking

品牌咨询完成后，如果用户尚未到店，自动引导车位预留：

```
Agent 文案追加：「如需驾车前来，我可以为您提前预留车位。」
快捷回复追加：[帮我预留车位]
```

### 8.3 store-consult → membership

非会员咨询品牌时，在回复末尾插入场景化权益引导：

```
Agent 文案追加：「SKP 黑卡会员可享品牌新品优先预览权与专属 SA 预约通道，是否需要为您办理入会？」
```

> 注意：仅插入一次，避免同一会话中重复引导。通过 `userProfile.isMember` 判断。

## 9. 与主 PRD 的关联

本功能对应《北京SKP AI私享管家系统产品需求文档 v1.0》§2.4.1 跨品牌高级穿搭与礼品顾问，同时延伸至来店前决策场景。

### 与主 PRD 的差异说明

| 主 PRD 要求 | 当前方案 | 后续规划 |
|-------------|---------|---------|
| 接入多品牌联合 ERP 库存系统 | ❌ 使用品牌知识库（静态 md） | M2 对接 ERP 库存接口 |
| 生成搭配建议图文 | ❌ 仅文本+品牌卡片 | M2 图文搭配方案 |
| 一键连接专柜 SA 留货 | ⚠️ 提供快捷回复入口 | M1 对接 SA 企微/分机 |
| 品牌实时库存查询 | ❌ 静态知识库 | M2 ERP 对接 |
