## 1. 功能概述

智能停车功能让用户通过自然语言对话完成停车位置记录与停车状态查询，无需手动操作任何表单或跳转页面。Agent 在用户告知停车位置后自动记录，后续用户随时询问即可获得停车位置、累计时长与费用计算。

**核心价值**：将传统"打开App → 找停车模块 → 手动输入车位号"的繁琐流程，压缩为一句自然语言交互。

## 4. 意图识别规则

### 4.1 停车位置录入

用户消息中需同时满足：
- **未匹配停车查询意图**（见 4.2）
- **包含可识别的位置信息**，支持以下格式：

| 输入格式 | 示例 | 解析结果 |
|----------|------|----------|
| `B<楼层>-<柱号>` | B2-D05、B3-A12 | floor="B2", location="B2-D05" |
| `地下<X>层 <柱号>` | 地下3层 B12 | floor="B3", location="B3-B12" |
| `停在/车在 + 柱号` | 停在C08、车位D05 | floor="B2"(默认), location="C08"/"D05" |

**解析优先级**：`B楼层-柱号` > `地下X层` > `停在+柱号`

### 4.2 停车状态查询

用户消息中包含停车相关关键词，且**不包含具体位置信息**：

**匹配关键词**：`停车`、`车位`、`车停`、`车在`、`停车费`、`停车费多少`、`停了多久`、`停车时长`

**排除条件**（命中以下模式则走录入流程而非查询）：
- `停[在到][B-Zb-z]`（如"停在B2-D05"）
- `停[在到]地下`（如"停在地下3层"）
- `停[在到]\d`（如"停在05"）

### 4.3 意图路由优先级

```
停车位置录入（含可解析位置）> 停车状态查询 > 兜底通用回复
```

## 5. 数据模型

### 5.1 前端状态

```typescript
interface ParkingInfo {
  location: string;   // 停车位置标识，如 "B2-D05"
  floor: string;       // 楼层标识，如 "B2"
  parkedAt: number;    // 入场时间戳（毫秒），用于计算时长和费用
}
```

- 通过 React `useState` 在组件内维护
- 当前为会话级记忆（刷新后丢失），M2 阶段将持久化至后端

### 5.2 消息扩展

```typescript
interface ParkingCard {
  type: "parking-card";
  location: string;    // 展示用位置文本，如 "B2层 · B2-D05"
  floor: string;       // 楼层
  duration: string;    // 时长文本，如 "2小时15分钟"
  fee: string;         // 费用金额，如 "30"
  feeRate: string;     // 费率说明，如 "15元/小时 · 前30分钟免费"
}
```

作为 `Message` 的可选字段 `parkingCard` 传递，与 `card`（会员卡）并列。

## 7. 费用计算规则

| 规则 | 说明 |
|------|------|
| 费率 | 15元/小时 |
| 免费时长 | 前30分钟免费 |
| 计费方式 | 超出免费时段后，按小时向上取整计费 |
| 计算公式 | `fee = Math.ceil((总分钟数 - 30) / 60) × 15`（≤30分钟时fee=0） |

> **注意**：当前费率为演示用固定值，M1 阶段将对接商场真实停车系统费率接口。

### 计算示例

| 停放时长 | 计费分钟 | 计费小时（向上取整） | 费用 |
|----------|----------|----------------------|------|
| 20分钟 | 0 | 0 | ¥0 |
| 45分钟 | 15 | 1 | ¥15 |
| 1小时30分钟 | 60 | 1 | ¥15 |
| 2小时15分钟 | 105 | 2 | ¥30 |
| 3小时01分钟 | 151 | 3 | ¥45 |
