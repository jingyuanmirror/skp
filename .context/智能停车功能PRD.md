## 1. 功能概述

智能停车功能让用户通过自然语言对话完成停车位置记录、停车状态查询与来店前车位预留，无需手动操作任何表单或跳转页面。Agent 在用户告知停车位置后自动记录，后续用户随时询问即可获得停车位置、累计时长与费用计算；用户来店前可提前预留车位，消除"有没有车位"的到店焦虑。

**核心价值**：
1. **到店后**：将传统"打开App → 找停车模块 → 手动输入车位号"的繁琐流程，压缩为一句自然语言交互
2. **来店前（v2.0 新增）**：通过车位预留消除"有没有车位/能不能停到好位置"的最大到店阻力

## 4. 意图识别规则

### 4.1 停车位置录入

用户消息中需同时满足：
- **未匹配停车查询意图**（见 4.2）
- **包含可识别的位置信息**，支持以下格式：

| 输入格式 | 示例 | 解析结果 |
|----------|------|----------|
| `B<楼层>-<柱号>` | B2-D05、B3-A12 | floor="B2", location="B2-D05" |
| `地下<X>层 <柱号>` | 地下3层 B12 | floor="B3", location="B3-B12" |
| `停在/车在 + 柱号` | 停在C08、车位D05 | floor="B2"(默认), location="C08"/"D05" |

**解析优先级**：`B楼层-柱号` > `地下X层` > `停在+柱号`

### 4.2 停车状态查询

用户消息中包含停车相关关键词，且**不包含具体位置信息**：

**匹配关键词**：`停车`、`车位`、`车停`、`车在`、`停车费`、`停车费多少`、`停了多久`、`停车时长`

**排除条件**（命中以下模式则走录入流程而非查询）：
- `停[在到][B-Zb-z]`（如"停在B2-D05"）
- `停[在到]地下`（如"停在地下3层"）
- `停[在到]\d`（如"停在05"）

### 4.3 意图路由优先级

```
车位预留（含预留/留车位关键词）> 停车位置录入（含可解析位置）> 停车状态查询 > 兜底通用回复
```

### 4.4 车位预留意图识别（v2.0 新增）

用户消息中包含预留关键词，且不包含具体停车位置（与"停在B2-D05"录入区分）：

**匹配关键词**：`预留|留车位|帮留|预留车位|留个车位|提前留|车位预留`

**正则表达式**：`/预留|留车位|帮留|预留车位|留个车位|提前留|车位预留/`

**楼层偏好提取**：

| 输入 | 提取结果 |
|------|----------|
| 帮我预留一个车位 | 楼层=自动分配 |
| 帮我留个B2的车位 | 楼层=B2 |
| 我想要B1的车位 | 楼层=B1 |

**预留时间提取**：

| 输入 | 提取结果 |
|------|----------|
| 我下午2点到，帮我预留 | 有效期至=14:15（到达时间+15分钟缓冲） |
| 帮我预留一个车位 | 有效期至=当前时间+60分钟（默认1小时） |
| 晚上7点到 | 有效期至=19:15 |

## 5. 数据模型

### 5.1 前端状态

```typescript
interface ParkingInfo {
  location: string;        // 停车位置标识，如 "B2-D05"
  floor: string;           // 楼层标识，如 "B2"
  parkedAt: number;        // 入场时间戳（毫秒），用于计算时长和费用
  // v2.0 新增：车位预留相关字段
  reservedSpot?: string;   // 预留车位号，如 "B2-R12"
  reservedUntil?: number;  // 预留到期时间戳（毫秒）
  reservationId?: string;  // 预约凭证号，如 "PK-20260629-R12"
  isVipSpot?: boolean;     // 是否 VIP 专属车位
}
```

- 通过 React `useState` 在组件内维护
- 当前为会话级记忆（刷新后丢失），M2 阶段将持久化至后端
- v2.0 新增字段仅在来店前车位预留场景使用；到店后记录实际停车位置时，预留字段清空

### 5.2 消息扩展

```typescript
interface ParkingCard {
  type: "parking-card";
  location: string;         // 展示用位置文本，如 "B2层 · B2-D05"
  floor: string;            // 楼层
  duration: string;         // 时长文本，如 "2小时15分钟"
  fee: string;              // 费用金额，如 "30"
  feeRate: string;          // 费率说明，如 "15元/小时 · 前30分钟免费"
  // v2.0 新增：预留模式
  mode?: "recorded" | "reserved";  // 卡片模式：已记录 / 已预留，默认 "recorded"
  reservedUntil?: string;   // 预留到期时间文本，如 "14:15"
  reservationId?: string;   // 预约凭证号，如 "PK-20260629-R12"
  isVipSpot?: boolean;      // VIP 车位标记
}
```

作为 `Message` 的可选字段 `parkingCard` 传递，与 `card`（会员卡）并列。

### 5.3 车位预留凭证号生成规则（v2.0 新增）

格式：`PK-{日期YYYYMMDD}-{车位号}`

示例：`PK-20260629-R12`、`PK-20260629-V03`

### 5.4 预留车位分配规则（Demo 阶段）

| 场景 | 分配逻辑 |
|------|---------|
| 用户指定楼层 | 在指定楼层分配随机车位号（格式：{楼层}-{R+2位随机数}） |
| 用户未指定楼层 | 默认分配 B2 层 |
| 钻卡/黑卡会员 | 优先分配 B1 层 VIP 车位（格式：B1-{V+2位随机数}），如 B1-V03 |
| 银卡会员 | 分配普通车位 |

### 5.5 预留有效期规则

| 情况 | 有效期计算 |
|------|----------|
| 用户指定到达时间（如"下午2点到"） | 到达时间 + 15 分钟缓冲 |
| 用户未指定时间 | 当前时间 + 60 分钟（默认 1 小时） |
| 上限 | 最长预留 3 小时 |

## 7. 费用计算规则

| 规则 | 说明 |
|------|------|
| 费率 | 15元/小时 |
| 免费时长 | 前30分钟免费 |
| 计费方式 | 超出免费时段后，按小时向上取整计费 |
| 计算公式 | `fee = Math.ceil((总分钟数 - 30) / 60) × 15`（≤30分钟时fee=0） |

> **注意**：当前费率为演示用固定值，M1 阶段将对接商场真实停车系统费率接口。

### 计算示例

| 停放时长 | 计费分钟 | 计费小时（向上取整） | 费用 |
|----------|----------|----------------------|------|
| 20分钟 | 0 | 0 | ¥0 |
| 45分钟 | 15 | 1 | ¥15 |
| 1小时30分钟 | 60 | 1 | ¥15 |
| 2小时15分钟 | 105 | 2 | ¥30 |
| 3小时01分钟 | 151 | 3 | ¥45 |

## 9.5 Skill 间联动设计（v2.0 新增）

### 9.5.1 parking ← navigation

导航 skill 查询路况后，如果用户选择驾车，自动提议车位预留：

```
navigation 弹出导航卡片 → 快捷回复包含 [帮我预留车位]
用户点击 → 路由到 parking 预留功能
```

### 9.5.2 parking ← appointment

品牌预约成功后，自动提议车位预留：

```
appointment 弹出预约卡片 → Agent 文案追加"同时为您预留车位？"
用户点击 [帮我预留车位] → 路由到 parking 预留功能
预留时间自动按预约时间倒推（预约 14:00 → 车位预留至 13:45）
```

### 9.5.3 parking → membership

非会员请求车位预留时，前置入会引导：

```
Agent：车位预留需开通 SKP 会员。钻卡会员可享 4 小时免费停车 + VIP 专属停车位预留权。
快捷回复：[我想入会] [了解会员权益] [稍后再说]
```
