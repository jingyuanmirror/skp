# 北京SKP AI私享管家系统产品需求文档 (PRD) v1.0

> **项目定位**：面向北京SKP高净值客群的“24小时随身高级私享数字管家”，全面重塑传统小程序的被动服务模式，打造高奢感、拟人化、具有场景感知能力的AI Agent，实现“服务找人”[cite: 1, 2]。
> **产品负责人**：Mirror (Big Tech Product Architect)[cite: 1, 2]
> **创建日期**：2026-06-26[cite: 1, 2]
> **当前状态**：草案评审中[cite: 1, 2]

---

## 1. 项目概述

### 1.1 背景与痛点分析
北京SKP作为全球顶级的高端奢华百货，拥有极高净值的会员客群[cite: 1, 2]。然而，现有的商场传统数字化工具（如微信小程序、App）仍处于传统的“货架式、人找服务”阶段，存在以下核心痛点[cite: 1, 2]：
* **操作冗长与用户阶层错配**：高净值客户（特别是黑卡、钻石卡会员）时间成本极高，对于繁琐的“绑定车牌-寻找停车模块-手动输入车位”或“繁琐表单入会”等操作耐受度极低[cite: 1, 2]。
* **营销低效与信息过载**：传统小程序常采用通铺型领券中心或盲目Push，缺乏场景与情绪感知，对高奢客户造成打扰，难以有效拉动跨品类连带消费[cite: 1, 2]。
* **服务资源瓶颈**：SKP顶级的1对1人工SA（导购）与专属客服资源有限，无法覆盖全量白金卡及银卡用户，导致中高端用户体验出现断层[cite: 1, 2]。

### 1.2 核心目标

| 维度 | AI Agent 智能体解决方案 | 核心业务指标 (KPI) |
| :--- | :--- | :--- |
| **消费者体验** | 打造7x24小时全天候、秒级响应、具备高奢语调的“随身管家”，提供无感入会、智能寻车及托管排队[cite: 1, 2]。 | 会员净推荐值 (NPS) 提升 35%<br>客服高频重复问询降低 90%[cite: 1, 2] |
| **促进商业消费** | 通过跨品类心愿单与动态“卡券策略引擎”，在精准场景（如排队、离店前夕）主动发放高相关性权益[cite: 1, 2]。 | 联带消费率提升 22%<br>客单价提升 15%[cite: 1, 2] |
| **商场管理能力** | 活化点击数据为动态语义意图数据，后台自动化聚类生成消费者趋势看板，反哺商场MD（招商调改）[cite: 1, 2]。 | 新增小众品牌意向捕获率 100%<br>SA线索转化率提升 40%[cite: 1, 2] |

## 2. 核心功能需求（功能清单）

本系统核心分为四大基石模块：拟人化无感入会、场景驱动智能停车、动态权益与卡券引擎、进阶私享管家服务[cite: 1, 2]。

### 2.1 模块一：拟人化无感入会 (Onboarding)
#### 2.1.1 业务流程与交互规范
系统拒绝长表单填报，采用全对话流引导[cite: 1, 2]。交互支持微信手机号一键授权绑定，并进行画像初始化[cite: 1, 2]：
1. 用户发送指令（语音或文字）：“帮我办张卡” 或 “怎么注册会员”[cite: 1, 2]。
2. 智能体自动调用微信凭证获取手机号，秒级完成后台CRM注册[cite: 1, 2]。
3. 智能体使用符合高奢调性的文案欢迎入会，并追问一句（不超过2个选择项）：“尊敬的会员，为了往后能为您精准预留新品，请问您平日更倾向关注：[高奢腕表/皮具]、[美妆护肤] 还是 [先锋设计师品牌]？”[cite: 1, 2]

### 2.2 模块二：场景驱动智能停车 (Smart Parking)
#### 2.2.1 空间感知与车位自动录入
* **图片识别录入**：允许用户直接拍照发送地库车位照片[cite: 1, 2]。Agent需通过视觉大模型（VLM）自动提取立柱编码（如：B3-D05），并调用停车场API与当前车牌号完成绑定[cite: 1, 2]。
* **语音录入**：用户发送语音“我停在地下三层B12柱子旁边”，Agent通过NLP提取“B3层”和“B12”，转换为标准化结构数据写入停车系统[cite: 1, 2]。

#### 2.2.2 主动计费与离场引导
当用户在对话中触发“几点关门”、“今天消费了多少”或位置定位接近地库时，Agent触发**主动推演**[cite: 1, 2]。
> **交互示例**
> **Agent**：“尊敬的李先生，检测到您的爱车已在B3-D05车位停放3小时40分钟[cite: 1, 2]。您今日累计消费已自动为您折算积分，是否需要我现在为您直接扣除15元停车费，并为您开启离场闸机绿色通道？”[cite: 1, 2]

### 2.3 模块三：动态权益与卡券策略引擎 (Smart Couponing)
拒绝静态“领券中心”，全面采用基于上下文、地理位置及历史画像的**动态即时策略算力发券**[cite: 1, 2]。

| 触发场景 | 用户语义/行为输入 | 大模型推演逻辑 | Agent主动服务动作 | 优先级 |
| :--- | :--- | :--- | :--- | :--- |
| **热门排队等待** | “香奈儿现在排队人多吗？”[cite: 1, 2] | 检测到对应品牌排队时长 > 15分钟，用户处于空闲等待状态[cite: 1, 2]。 | 告知排队时间，并精准下发商场4层高奢咖啡厅的“85折会员专属咖啡券”，引导客户错峰等待[cite: 1, 2]。 | **高 (P0)**[cite: 1, 2] |
| **跨品类连带** | “刚买了包包，推荐个吃饭的地方”[cite: 1, 2] | 识别用户刚完成高客单价消费，根据CRM画像其偏好黑珍珠餐厅[cite: 1, 2]。 | 推荐商场内“新荣记”，并主动提示：“由于您是钻石卡会员，已为您申请今日优先入座权，是否帮您安排？”[cite: 1, 2] | **中 (P1)**[cite: 1, 2] |
| **离店临界挽回** | “BHG超市几点打折？”或查询停车费[cite: 1, 2] | 识别到用户行程即将结束，准备离开商场[cite: 1, 2]。 | 精准下发一张“满500减80”的晚间高端餐饮定向满减券，尝试留存用户在商场享用晚餐[cite: 1, 2]。 | **高 (P0)**[cite: 1, 2] |

### 2.4 模块四：进阶私享管家服务
#### 2.4.1 跨品牌高级穿搭与礼品顾问
Agent需接入SKP多品牌联合ERP库存系统[cite: 1, 2]。当用户提出复杂长尾的需求时，提供一体化方案[cite: 1, 2]：
* **用户输入**：“今晚参加一个商务晚宴，穿深色西装，推荐配什么丝巾？要SKP店里现在就能买到的。”[cite: 1, 2]
* **Agent动作**：检索今日在库的高奢品牌（如Hermès、Gucci）丝巾新品，生成搭配建议图文，并提供一键连接专柜SA（导购）留货的按钮[cite: 1, 2]。

#### 2.4.2 全功能排队托管
深度打通商场内所有餐饮、潮牌限制人流专柜的排队系统[cite: 1, 2]。用户发出指令“帮我排一个新荣记3人位”，由Agent在后台代为持有号牌，并在预计到号前10分钟，通过语音和微信服务号主动提醒用户前往[cite: 1, 2]。

## 3. 非功能性需求与专家视点

### 3.1 高奢语气管理规范 (Tone of Voice)
大模型Prompt必须经过严格的意识形态与语调对齐[cite: 1, 2]。全面禁止出现任何传统电商或下沉平台的客服用语[cite: 1, 2]：
* **禁用词**：“亲”、“亲亲”、“家人们”、“小仙女”、“收到宝贝”等[cite: 1, 2]。
* **标准用语**：应采用“尊敬的SKP会员”、“很高兴为您服务”、“已为您妥善安排”等[cite: 1, 2]。语气需保持谦逊、专业、具有边界感和高级感[cite: 1, 2]。

### 3.2 数据安全与合规性
* **隐私绝对保护**：严禁AI Agent在未获授权情况下将用户的敏感消费数据、车牌数据、面部数据泄露给第三方品牌商[cite: 1, 2]。
* **严格控制幻觉 (Zero-Hallucination RAG)**：涉及品牌具体专柜位置、商场活动满减克扣标准、积分扣减比例，AI绝对不允许自我捏造[cite: 1, 2]。必须采用**强限制RAG（检索增强生成）**架构，凡在商场官方知识库中未检索到的数据，AI统一话术回复：“正在帮您联系商场人工台核实”[cite: 1, 2]。

## 4. 系统架构与集成接口

为保障上述体验的顺畅落地，系统必须与SKP现有的数字化基建深度打通，接口规划如下[cite: 1, 2]：

| 接口名称 | 对接系统 | 数据传输方向 | 核心传输内容 |
| :--- | :--- | :--- | :--- |
| **会员鉴权与画像接口** | 商场官方 CRM 系统 | 双向 (Bidirectional)[cite: 1, 2] | 会员等级（黑卡/钻石/银卡）、剩余积分、历史高频消费品类标签[cite: 1, 2]。 |
| **车牌与地库感知接口** | 智能停车场管理系统 | 双向 (Bidirectional)[cite: 1, 2] | 车牌绑定状态、入场时间、车位编号（立柱号）、应付停车费结算、积分抵扣核销[cite: 1, 2]。 |
| **即时库存(库存检索)接口** | 商场联立 ERP 系统 | 只读 (Read Only)[cite: 1, 2] | 各奢品专柜今日在库SKU、尺码状态、专柜具体分机/SA企微联系方式[cite: 1, 2]。 |
| **营销与动态发券接口** | 商场 POS/券包系统 | 双向 (Bidirectional)[cite: 1, 2] | 动态优惠券ID生成、核销状态同步、定向满减券实时下发[cite: 1, 2]。 |

## 5. 落地规划与里程碑 (Milestones)

由于高奢百货对数字化工具的容错率极低，建议采取渐进式敏捷开发[cite: 1, 2]：
1. **M1 基础体验上线（第1-2个月）**：上线高奢语调客服、一句话入会（打通CRM）、拍照记车位与自助积分算费[cite: 1, 2]。
2. **M2 主动推荐引擎（第3-5个月）**：打通地理位置感知与即时排队系统，上线基于场景的主动发券机制和托管排队功能[cite: 1, 2]。
3. **M3 商业智能与时尚买手（第6个月以上）**：全面上线多品牌ERP库存联动的时尚穿搭AI顾问，管理后台上线针对用户语义聚类的MD招商决策分析看板[cite: 1, 2]。
