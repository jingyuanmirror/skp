## 1. 功能概述

智能导航功能让用户在来店途中通过自然语言查询路况、推荐路线、预估到店时间，并获得停车场入口推荐和室内导航提示。Agent 结合用户出发地与目的地品牌楼层，提供从"出门"到"进店"的一站式导航服务。

**核心价值**：消除"会不会堵车/找不到路/找不到车位"的来店阻力，把用户从"考虑来不来"推到"已在路上"。

## 4. 意图识别规则

### 4.1 navigation 意图

| 子意图 | 匹配模式 | 示例 |
|--------|---------|------|
| 路况查询 | 出发地 + 堵/路况/多久 | "从三里屯过来堵不堵"、"开车过来多久" |
| 路线推荐 | 怎么去/怎么走/如何到 | "怎么来 SKP 最快"、"去SKP怎么走" |
| 地铁查询 | 地铁/几号线/坐地铁 | "地铁几号线到 SKP"、"坐地铁怎么去" |
| 停车入口 | 停车入口/从哪进/哪个门 | "SKP 停车从哪个入口进"、"去 1F 从哪进最近" |

### 4.2 intentDescription（供 LLM 分类器使用）

```
navigation: 用户查询来SKP的路线、驾车路况、地铁交通、停车入口推荐、室内导航等出行导航场景
```

### 4.3 出发地提取

Agent 需从用户消息中提取出发地关键词：

| 输入 | 提取结果 |
|------|----------|
| 从三里屯过来 | 出发地=三里屯 |
| 望京过来 | 出发地=望京 |
| 从国贸 | 出发地=国贸 |
| 机场过来 | 出发地=机场 |
| 开车过来 / 怎么来 | 出发地=未知（提供通用路线） |

## 5. 数据模型

### 5.1 导航知识库（`navigation-routes.md`）

```markdown
## 常用驾车路线

### 三里屯 → SKP
- 路线：东三环北路 → 建国路
- 正常：20 分钟
- 高峰：30-40 分钟
- 推荐入口：B1 西侧

### 望京 → SKP
- 路线：四环 → 京通快速 → 建国路
- 正常：25 分钟
- 高峰：35-45 分钟
- 推荐入口：B2 东侧

## 地铁信息
- 1号线 大望路站 B口出 → 向东 200 米 → SKP 正门
- 10号线 国贸站换乘 1号线 → 大望路站 B口

## 停车场入口与楼层映射
- B1 西侧入口 → 适合 1F 西侧品牌（Chanel、Dior）
- B1 东侧入口 → 适合 1F 东侧品牌（Gucci、LV）
- B2 西侧入口 → 适合 2F 品牌
- B2 东侧入口 → 适合 B1 美食层 + 3F 品牌
- B3 入口 → 适合长期停车

## 室内导航（电梯与品牌分布）
- 西侧电梯 → 1F Chanel/Dior, 2F 成衣区, 3F 珠宝区
- 东侧电梯 → 1F Gucci/LV/Hermès, 2F 户外时尚
- 中庭电梯 → 全楼层，适合 4F-7F 餐饮层
```

### 5.2 NavigationCard 数据结构

```typescript
interface NavigationCard {
  type: "navigation-card";
  origin?: string;           // 出发地，如 "三里屯"（通用路线时省略）
  destination: string;       // 目的地，如 "北京 SKP"
  mode: "drive" | "transit" | "compare";  // 驾车/地铁/多方案对比
  routes: NavigationRoute[];
  recommendedEntrance?: ParkingEntrance;  // 推荐停车入口
}

interface NavigationRoute {
  mode: "drive" | "metro" | "taxi";
  duration: string;          // 如 "25分钟"
  description: string;       // 如 "东三环北路→建国路"
  status?: string;           // 路况状态，如 "轻微拥堵"
  cost?: string;             // 打车费用，如 "¥35"
}

interface ParkingEntrance {
  entrance: string;          // 如 "B1 西侧入口"
  targetFloor: string;       // 适合楼层，如 "1F"
  note: string;              // 如 "乘西侧电梯至1F，右转50米"
}
```

### 6.4 设计要点

- **头部**：暖色渐变 + 「🧭 智能导航」标识 + 模式标签
- **路线区**：起点→终点一行显示，Cormorant 字体
- **方案列表**：图标 + 时间 + 补充说明（路况/站点/费用），每行一个方案
- **路况状态色**：🟢 畅通 / 🟡 轻微拥堵 / 🔴 拥堵
- **停车入口**：🅿️ 图标 + 入口名 + 简要导航提示

## 8. Skill 间联动设计

### 8.1 navigation → parking

路况查询后，如果用户选择驾车，自动提议车位预留：

```
Agent 文案追加：「需要我为您预留车位吗？预留后到店即可停入指定位置。」
快捷回复追加：[帮我预留车位]
```

### 8.2 navigation → store-consult

停车场入口推荐与品牌位置信息联动：

```
用户指定品牌 → navigation 查询停车入口 → 弹出导航卡片
快捷回复追加：[帮我在{品牌}预约档期] [联系专属SA]
```

### 8.3 navigation → membership

非会员查路况时插入会员引导：

```
Agent 文案追加：「开通 SKP 会员可享车位预留服务，到店即停无需找位，是否需要为您办理入会？」
```

## 9. 与主 PRD 的关联

本功能为新增模块，对应来店前动线中的"来店途中导航"环节，主 PRD 尚未覆盖此场景。

### 与主 PRD 的差异说明

| 主 PRD 要求 | 当前方案 | 后续规划 |
|-------------|---------|---------|
| 实时路况 API 集成 | ❌ 使用知识库静态数据+模拟 | M1 对接高德/百度地图 API |
| 室内导航系统 | ⚠️ 文字提示级导航 | M2 AR 室内导航 |
| 打车集成 | ❌ 仅文字建议 | M2 对接滴滴/高德打车 |
| 位置实时感知 | ❌ 无 | M2 GPS 定位集成 |
