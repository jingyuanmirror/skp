## 1. 功能概述

奢侈品品牌预约功能让用户通过自然语言直接发起品牌专柜的档期预约，无需到店排队等待。与餐厅排号（`queue`）的"先到先排"被动模式不同，品牌预约是"选档期预约"的主动模式——用户指定品牌、时间段和人数后，Agent 生成预约凭证，锁定专属档期和 SA，并在预约时间前主动提醒。

**核心价值**：将奢侈品到店体验从"到店排队"升级为"预约即享"，覆盖"我想去香奈儿，帮我预约一个档期"的自然语言直达场景。

---

## 2. 与 queue 的关系与区分

| 维度 | queue（餐厅排号） | appointment（品牌预约） |
|------|-------------------|--------------------------|
| **模式** | 被动：先到先排，到场后取号 | 主动：选档期预约，提前锁定 |
| **时间性** | 即时取号，等位不可控 | 选择具体时段，确认后锁定 |
| **用户意图** | "帮我排一个号" | "帮我约一个档期" |
| **号牌逻辑** | 排队号 + 前方等待组数 + 倒计时 | 预约凭证 + 档期确认 + SA 信息 |
| **提醒机制** | 即将到号 → 已到号 | 预约前 30 分钟提醒出发 |
| **适用场景** | 餐厅、热门品牌限流排队 | 奢侈品品牌 SA 专属预约 |
| **关键词** | 排号/排队/帮我排 | 预约/约档期/帮我约 |

**路由规则**："排号/排队"关键词 → `queue`；"预约/约档期"关键词 → `appointment`。LLM 分类器辅助区分模糊意图。

## 5. 意图识别规则

### 5.1 appointment 意图

| 子意图 | 匹配模式 | 示例 |
|--------|---------|------|
| 预约创建 | 品牌 + 预约/约/档期 | "帮我预约Chanel"、"我想去香奈儿约个档期" |
| 档期查询 | 品牌 + 哪天/有档期/能约 | "周末 LV 有档期吗"、"Hermès 今天还能约吗" |
| 预约状态查询 | 我的预约/约的几点 | "我的预约几点"、"我约的Chanel几点" |
| 预约管理 | 取消/改期/换时间 | "帮我取消预约"、"我想改个时间" |

### 5.2 intentDescription（供 LLM 分类器使用）

```
appointment: 用户想要预约奢侈品品牌专柜档期、查询可预约时段、管理已有预约（取消/改期）、查询预约状态等品牌预约场景
```

### 5.3 与 queue 的路由区分

| 用户输入 | 路由目标 | 判断依据 |
|---------|---------|---------|
| "帮我排一个新荣记3人位" | queue | "排"+ 餐厅名 |
| "帮我排Chanel" | queue | "排"+ 品牌名（"排"优先路由到 queue） |
| "帮我预约Chanel" | appointment | "预约"关键词 |
| "我想去香奈儿，帮我约一个档期" | appointment | "约/档期"关键词 |
| "Chanel排队多久" | cross-sell | "排队多久"= 查等候时长 |

**优先级**：appointment 关键词（预约/约档期）> queue 关键词（排号/排队）> cross-sell 关键词（排队多久）

## 8. 预约提醒机制

### 8.1 提醒策略

| 事件 | 触发条件 | Agent 行为 |
|------|----------|-----------|
| 出发提醒 | 预约时间前 30 分钟 | 发送消息：「您的预约将在 30 分钟后开始，请合理安排出发时间」+ 快捷回复 |
| 即将开始提醒 | 预约时间前 10 分钟 | 发送消息：「您的预约即将开始，请前往{楼层}」+ 快捷回复 |

> 提醒逻辑复用 queue 的定时器机制，但触发时间点和内容不同。

### 8.2 App.tsx 集成

- `appointmentInfo` 加入 App 状态管理
- 预约创建后启动 30 分钟/10 分钟定时器
- 定时器触发时自动注入提醒消息到聊天流
- 预约取消/完成后清除定时器

## 10. Skill 间联动设计

### 10.1 appointment → parking

预约成功后，自动提议车位预留：

```
Agent 文案追加：「预约时间前 30 分钟我会提醒您出发。同时为您预留车位？」
快捷回复追加：[帮我预留车位]
```

如果用户确认预留车位，按预约时间倒推预留生效时间（如预约 14:00，车位预留至 13:45 生效）。

### 10.2 appointment → coupon

预约成功后，推送品牌优惠券：

```
Agent 文案追加：「已为您附上 Chanel 会员专属优惠券，到店即可使用。」
Action：自动附加品牌券到 coupons 数组
```

### 10.3 appointment → membership

非会员发起预约时，前置入会引导：

```
Agent：品牌预约需开通 SKP 会员。钻卡及以上可享品牌优先预约权。
快捷回复：[我想入会] [了解会员权益] [稍后再说]
```

### 10.4 store-consult → appointment

品牌咨询完成后，如果用户表达试看/购买意愿，skill 路由自动切换到 appointment：

```
用户在 store-consult 对话后点击 [帮我预约Chanel] → 路由到 appointment
```

## 11. 与主 PRD 的关联

本功能为新增模块，从 `queue` 排号功能中独立拆分，对应奢侈品品牌预约的来店前场景。主 PRD §2.4.2 中"全功能排队托管"覆盖了餐厅排队，但未单独规划奢侈品品牌预约。

### 与主 PRD 的差异说明

| 主 PRD 要求 | 当前方案 | 后续规划 |
|-------------|---------|---------|
| 品牌专柜排队托管（先到先排） | ✅ 由 queue 覆盖 | M1 对接品牌限流系统 |
| 品牌 SA 预约（选档期预约） | ✅ 由 appointment 覆盖 | M1 对接 CRM SA 排班系统 |
| SA 企微一键连接 | ❌ 仅展示 SA 姓名 | M1 对接 SA 企微/分机 |
| 预约档期实时查询 | ❌ 静态知识库 | M1 对接品牌预约系统 API |
| 多品牌同时预约 | ❌ 单品牌预约 | M2 多品牌并行预约 |
| 预约数据持久化 | ❌ 会话级 | M1 后端持久化 |
